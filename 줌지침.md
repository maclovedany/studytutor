# Zoom 미팅 임베드 + 15분 결제 게이트 — 구현 지침

> 대상 범위: PRD 9.9(상담예약/Zoom) / 9.10(15분 타이머) / 9.11(데모 결제)의 강화판
> 핵심 목표: **상담 미팅을 웹페이지 안에 임베드하고, 무료 15분이 끝나면 통화에서 자동 퇴장 → 결제하면 같은 미팅에 자동 재입장**
> 스택: Next.js 16, Supabase, Zoom(Server-to-Server OAuth + Meeting SDK), 결제 = 데모

---

## 0. 핵심 개념 — 왜 "임베드"인가

이전 구조의 문제: Zoom 링크를 **외부 앱으로** 열어주면, 사용자는 Zoom 화면만 보고 있어서
웹페이지에 뜨는 "15분 종료 → 결제하세요" 모달을 **못 본다**.

```
[이전]  웹페이지(타이머·모달)  ←  서로 별개  →  Zoom 앱(영상)
[현재]  웹페이지 안에 [영상 + 타이머 + 모달] 전부 한 화면
```

임베드하면 15분이 되는 순간 **영상이 있던 자리에서 바로** 퇴장·모달·결제·재입장이 일어난다.

또 하나의 핵심: **서버 강제 게이트**. 타이머는 브라우저에서 돌지만, 미팅에 입장하려면
서버가 발급하는 "입장 서명"이 필요하다. 서버는 "무료회원 & 15분 경과 & 미결제"면
서명을 거부(403)하므로 **새로고침으로는 절대 우회할 수 없다.**

## 1. 전체 흐름

```
[예약]  POST /api/consultations
        └─ Zoom 설정됨? → S2S OAuth 토큰 → 실제 미팅 생성
                          → consultations 에 zoom_meeting_id / zoom_password / zoom_url 저장
        └─ 미설정/실패?  → 기존 데모 링크(DEMO_ZOOM_URL)로 폴백 (예약은 절대 실패 안 함)

[시작]  "상담 시작하기" → started_at 저장 → Room 이 <ZoomMeeting> 마운트
        └─ POST /api/zoom/signature → 서버 게이트 통과 시 서명 반환 → SDK init/join

[15:00] (무료회원) shouldShowPaywall = true
        └─ Room 이 <ZoomMeeting> 언마운트 → SDK leaveMeeting() = 통화 자동 퇴장
        └─ 결제 모달 표시 (₩9,900 데모 결제)

[결제]  POST /api/payments/demo → tier = paid → router.refresh()
        └─ paywall false → <ZoomMeeting> 재마운트 → 새 서명(이번엔 paid 라 허용)
        └─ 같은 미팅에 자동 재입장 → 이어서 상담 🎉
```

## 3. 사전 준비 (다른 환경에서 재현 시 이것부터)

### 3-1. Zoom 콘솔에서 앱 2개 만들기 — [marketplace.zoom.us](https://marketplace.zoom.us) → Develop → Build App

| 앱 종류 | 용도 | 여기서 얻는 값 |
|---|---|---|
| **Server-to-Server OAuth** | 서버가 미팅을 생성 | `Account ID`, `Client ID`, `Client Secret` |
| **General App (Meeting SDK)** | 웹페이지 임베드 입장 서명 | `SDK Client ID`, `SDK Client Secret` |

- S2S OAuth 앱에는 **`meeting:write` 계열 스코프**를 추가하고 Activate 해야 한다.
- 두 앱은 서로 다른 자격증명이다. 섞으면 "Invalid signature" 나 401이 난다.
- 1:1 미팅은 Zoom **무료 플랜도 시간 제한 없음** → 비용 없이 테스트 가능.

### 3-2. 환경변수 5종 — `.env.local`

```bash
ZOOM_ACCOUNT_ID=           # S2S OAuth 앱
ZOOM_S2S_CLIENT_ID=
ZOOM_S2S_CLIENT_SECRET=
ZOOM_SDK_CLIENT_ID=        # Meeting SDK 앱
ZOOM_SDK_CLIENT_SECRET=
```

- **5개 모두 있어야** 임베드가 켜진다. 하나라도 없으면 자동으로 데모 링크 모드.
- 전부 **서버 전용**(NEXT_PUBLIC 아님). 클라이언트에는 서명 문자열만 내려간다.
- 넣은 뒤 dev 서버 재시작 필수.

### 3-3. DB 마이그레이션 — Supabase SQL Editor 에서 실행

```sql
alter table consultations add column if not exists zoom_meeting_id text;
alter table consultations add column if not exists zoom_password text;
```

(또는 `supabase/setup_all.sql` 전체 재실행 — 멱등이라 안전)

## 4. 동작 원리 상세

### 4-1. 미팅 생성 (예약 시, 서버)

```
POST https://zoom.us/oauth/token?grant_type=account_credentials&account_id=...
  Authorization: Basic base64(S2S_CLIENT_ID:S2S_CLIENT_SECRET)
  → access_token

POST https://api.zoom.us/v2/users/me/meetings
  Authorization: Bearer {access_token}
  body: { topic, type: 2, settings: { join_before_host: true, waiting_room: false } }
  → { id(미팅번호), password, join_url }
```

- `join_before_host: true` 가 중요 — MVP 에는 전문가(호스트) 계정이 없어서,
  호스트 없이도 참가자가 입장할 수 있어야 한다.
- 생성 실패 시 예약 라우트가 catch 해서 데모 링크로 폴백한다. **예약이 Zoom 장애로 죽지 않는다.**

### 4-2. 입장 서명 (JWT) — 왜 필요한가

Meeting SDK 는 아무나 join 못 하게, **서버가 Client Secret 으로 서명한 JWT** 를 요구한다.

```
클레임: { appKey: SDK_CLIENT_ID, sdkKey: 동일, mn: "미팅번호(문자열)",
          role: 0(참가자), iat, exp: iat+2h, tokenExp: exp }
서명:   HMAC-SHA256(base64url(header).base64url(payload), SDK_CLIENT_SECRET)
```

- `exp` 는 `iat + 1800초(30분)` 이상이어야 SDK 가 받아준다 (우리는 2시간).
- 별도 JWT 라이브러리 없이 `node:crypto` 로 직접 서명한다 (~15줄, `lib/zoom.ts`).

### 4-3. 서버 게이트 — 결제 강제의 핵심

`/api/zoom/signature` 는 서명을 주기 전에 검사한다:

```ts
if (shouldShowPaywall({ tier }, consultation.started_at, new Date()))
  return 403 { error: "payment_required" }
```

- `shouldShowPaywall` 은 클라이언트 타이머(Room)와 **같은 함수** (`lib/timer.ts`) →
  화면 표시와 서버 판정이 절대 어긋나지 않는다.
- 무료회원이 15분 지나서 새로고침해도, 서명이 안 나오므로 재입장 불가.
- 유료회원(paid)은 항상 통과.

### 4-4. 임베드 — 왜 npm 이 아니라 CDN 인가

- npm 패키지 `@zoom/meetingsdk@6.2.0` 은 **peer dependency 가 `react@18.2.0` 고정**이라
  이 프로젝트(React 19)에서 `npm install` 이 실패한다 (실측 확인).
- 그래서 자체 완결형 **CDN 전역 번들**을 런타임에 로드한다:
  `https://source.zoom.us/6.2.0/zoom-meeting-embedded-6.2.0.min.js` → `window.ZoomMtgEmbedded`
- 사용하는 API 4개 (패키지 타입 정의에서 실측 확인):
  ```ts
  const client = window.ZoomMtgEmbedded.createClient();
  await client.init({ zoomAppRoot: div요소, language: "ko-KR" });
  await client.join({ signature, meetingNumber, password, userName });
  await client.leaveMeeting();   // 퇴장
  ```

### 4-5. 퇴장/재입장 메커니즘 — React 마운트로 제어

Room.tsx 의 렌더 조건 한 줄이 전부다:

```tsx
{consultation.zoom_meeting_id && !paywall && (
  <Card><ZoomMeeting consultationId={consultation.id} /></Card>
)}
```

- 15분 도달 → `paywall` true → 컴포넌트 **언마운트** → cleanup 에서 `leaveMeeting()` = 자동 퇴장
- 결제 → `tier=paid` → `paywall` false → 컴포넌트 **재마운트** → 새 서명 fetch → 같은 미팅 재입장
- 별도의 타이머-SDK 연결 코드가 필요 없다. React 의 마운트 생명주기가 곧 입장/퇴장이다.

## 5. 데모 폴백 (Zoom 없이도 전부 동작)

| 상황 | 동작 |
|---|---|
| env 5종 미설정 | 예약 시 데모 링크 저장 → 기존과 동일(외부 링크 + 클라이언트 타이머) |
| 미팅 생성 API 실패 | 콘솔 경고 후 데모 링크 폴백, 예약 성공 유지 |
| 서명 403 (결제 필요) | 임베드 조용히 미표시 — 결제 모달이 안내 |
| SDK 로드/입장 실패 | "임베드를 불러오지 못했습니다" + 외부 링크로 상담 지속 가능 |
| 기존(마이그레이션 전) 상담 | `zoom_meeting_id` 없음 → 데모 링크 상담으로 취급 |

## 6. 테스트 방법

1. **15분 기다리지 말 것** — `lib/timer.ts` 의 `FREE_LIMIT_SECONDS = 15 * 60` 을
   잠깐 `60`(1분)으로 바꾸고 테스트 후 되돌린다.
2. env 설정 → dev 서버 재시작 → **새로 상담 예약** (기존 예약은 데모 링크 상담이라 임베드 안 뜸)
3. 상담 시작 → 페이지 안에 Zoom 화면이 뜨는지 확인
4. 1분 경과 → 자동 퇴장 + 결제 모달 확인
5. 새로고침해도 재입장이 안 되는지 확인 (**서버 게이트 검증** — 중요)
6. 데모 결제 → 같은 미팅 자동 재입장 확인
7. 자동 테스트: `npm test` (zoom 5개 포함 46개)

## 7. 자주 걸리는 문제

| 증상 | 원인/해결 |
|---|---|
| "Invalid signature" | SDK 앱의 Client ID/Secret 을 써야 함 (S2S 앱 값 아님). 서버 재시작 했는지 확인 |
| 미팅 생성 401/400 | S2S 앱 스코프(`meeting:write`) 누락 또는 앱 미활성화(Activate) |
| 예약은 되는데 임베드가 안 뜸 | env 5종 중 누락 → 데모 폴백 중. 또는 **마이그레이션 미실행**(0004_zoom.sql) |
| 임베드 자리가 비어 있음 | 브라우저 콘솔 확인 — CDN 차단(사내망/광고차단기) 가능. 외부 링크로 폴백 사용 |
| dev 에서 입장이 두 번 깜빡임 | React strict mode 의 effect 이중 실행 — 프로덕션에선 없음 |
| 15분 후에도 재입장됨 | `started_at` 이 없는 상담(시작 안 누름)은 게이트 통과가 정상. 시작 후인지 확인 |

## 8. 보안 유의사항

- Client Secret 2종은 **서버 파일(`lib/zoom.ts`)에서만** 사용. `"server-only"` import 로
  클라이언트 번들 유입을 빌드 타임에 차단한다.
- 서명 라우트는 ①로그인 ②본인 상담인지 ③결제 게이트, 3중 확인 후에만 서명을 준다.
- 서명 자체도 2시간이면 만료된다.

## 9. 현재 한계 / 후속 과제

- **전문가(호스트) 계정 없음** — 미팅은 플랫폼 계정 명의로 생성되고, 전문가는 `join_url` 로
  외부 입장하는 구조. 전문가 계정 체계는 후속.
- 15분 퇴장은 참가자(사용자) 쪽만 — 미팅 자체를 종료하지는 않는다.
  필요하면 Zoom API `PUT /meetings/{id}/status {action:"end"}` 로 강제 종료를 얹을 수 있다.
- 결제 유도 강도, 재입장 UX 문구 등은 실사용 데이터 보고 조정.
